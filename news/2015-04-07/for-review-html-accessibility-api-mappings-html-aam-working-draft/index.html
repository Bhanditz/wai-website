For Review: HTML Accessibility API Mappings (HTML-AAM) Working Draft

HTML Accessibility API Mappings 1.1 (HTML-AAM) is available for review as a First Public Working Draft. This specification defines how user agents map HTML markup to platform accessibility application programming interfaces (APIs). It provides HTML-specific guidance to define how the HTML user agent must respond to keyboard focus, native HTML features, and role, state, and property attributes provided in web content via WAI-ARIA. For more information, see the Call for Review: HTML Accessibility API Mappings (HTML-AAM) Working Draft e-mail. Please send comments by 8 May 2015.
